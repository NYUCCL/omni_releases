Optimizing Memory using Neural Information (OMNI)

Code Release 1:

Code Release 1 consists of code for estimating the two-state recall-only model in our Knowledge Tracing Using the Brain paper that we presented at Educational Data Mining 2018.

The code is available on github here: https://github.com/NYUCCL/omni_releases/tree/master/code_release_1.

The Stan code for the model can be found in the models folder. The only difference between the two files there is that one is setup to make use of Stan's new multithreading capacity.

The main files needed to run the code can be found in the shell_scripts folder. This contains three shell scripts for running the Stan models in batch mode on a slurm HPC cluster. run_two_state_logit_model_fmri_subs_kfcv.s divides the data from subjects who were in the fMRI into ten folds as described in the paper and estimates the models holding one fold out. run_two_state_logit_model_loocv.s estimates the models by holding one fMRI subject's data out. run_two_state_logit_parallel_model_loocv.s does the same thing but making use of parallelization. You can estimate all the models in parallel on your own cluster by calling sbatch --array=0-n run_two_state_logit_model_loocv.s . n should be the number of folds, i.e. 10 in the kfold case.
